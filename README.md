# 개혁주의 신앙 문답 챗봇 — 배포 안내서 (Google Gemini 버전)

이 버전은 **Google Gemini의 무료 API**를 사용합니다.
신용카드가 필요 없고, 하루 약 1,500건까지 무료이며, 한도는 매일 초기화됩니다.
청년부 챗봇으로는 차고 넘치는 양입니다.

## 폴더 구조

```
reformed-chatbot-gemini/
├─ api/
│  └─ chat.js          ← 서버리스 함수 (API 키를 안전하게 보관)
├─ public/
│  └─ index.html       ← 사용자가 보는 챗봇 화면
└─ package.json        ← 프로젝트 설정
```

원리: 사용자의 브라우저는 `index.html`만 봅니다. 실제 API 키는
`chat.js` 안에서만 쓰이고, 이 파일은 Vercel 서버에서만 실행되므로
사용자에게 절대 노출되지 않습니다.

---

## 준비물

1. **Google 계정** — 이미 가지고 계실 겁니다.
2. **GitHub 계정** — https://github.com (무료)
3. **Vercel 계정** — https://vercel.com (무료, GitHub으로 로그인)

신용카드는 어디에서도 필요하지 않습니다.

---

## 1단계 — Gemini API 키 발급 (카드 불필요)

1. https://aistudio.google.com 접속 → Google 계정으로 로그인
2. 왼쪽 메뉴 또는 화면에서 **Get API key** (API 키 가져오기) 클릭
3. **Create API key** (API 키 만들기) 클릭
4. 생성된 키를 복사해 둡니다. (`AIza...` 형태로 시작합니다)

> 이 키는 나중에 3단계에서 사용합니다. 안전한 곳에 임시로 메모해 두세요.
> 키는 누구에게도, 어떤 채팅에도 붙여넣지 마세요.

---

## 2단계 — GitHub에 코드 올리기

1. github.com 로그인 → 오른쪽 위 `+` → **New repository**
2. 이름은 `reformed-chatbot` 등으로 짓고 **Create repository**
3. **uploading an existing file** 링크 클릭
4. `reformed-chatbot-gemini` 폴더 안의 파일들을 **폴더 구조 그대로**
   끌어다 놓습니다 (`api` 폴더, `public` 폴더, `package.json`)
5. **Commit changes** 클릭

---

## 3단계 — Vercel에 연결하고 키 등록하기

1. vercel.com 에 GitHub 계정으로 로그인
2. **Add New...** → **Project** → 방금 만든 저장소 **Import**
3. 설정 화면에서 **Environment Variables** 항목을 펼칩니다.
4. 다음을 등록합니다 (이름을 정확히 입력하는 것이 가장 중요):

   | Name (이름)      | Value (값)                          |
   |------------------|-------------------------------------|
   | `GEMINI_API_KEY` | 1단계에서 발급받은 키 (`AIza...`)   |

   - 입력 후 **Add** 클릭

5. **Deploy** 버튼 클릭. 1~2분 후 완료됩니다.

> `GEMINI_MODEL` 변수는 따로 등록하지 않아도 됩니다.
> 등록하지 않으면 자동으로 `gemini-2.5-flash`를 사용합니다.
> 나중에 모델을 바꾸고 싶을 때만 이 변수를 추가하면 됩니다 (아래 참고).

---

## 4단계 — 확인하기

배포가 끝나면 Vercel이 주소를 줍니다 (예: `reformed-chatbot.vercel.app`).
그 주소로 들어가 질문해 보세요. 답변이 나오면 성공입니다.

---

## 문제가 생겼을 때 — 이 버전은 오류 메시지가 친절합니다

화면에 뜨는 메시지를 보고 바로 원인을 알 수 있습니다.

- **"API 키가 올바르지 않습니다"**
  → 3단계의 `GEMINI_API_KEY` 값이 잘못되었거나 공백이 섞였습니다.
     Vercel에서 값을 다시 입력한 뒤 **반드시 Redeploy** 하세요.

- **"모델 '...'을(를) 찾을 수 없습니다"**
  → 사용 중인 Gemini 모델이 교체되었습니다. 아래 '모델 교체' 참고.

- **"오늘의 무료 사용 한도를 초과했습니다"**
  → 드문 경우지만, 하루 한도를 다 썼습니다. 다음 날 자동 초기화됩니다.

- **"서버에 API 키가 없습니다"**
  → 환경 변수 이름이 `GEMINI_API_KEY`가 맞는지 확인하세요.

> 환경 변수를 고친 뒤에는 항상:
> Vercel → Deployments → 최신 항목 → ··· → **Redeploy**

---

## 모델 교체 (앞으로를 위한 대비)

AI 모델은 시간이 지나면 새 버전으로 교체될 수 있습니다.
만약 "모델을 찾을 수 없습니다" 오류가 뜨면, **코드를 고칠 필요 없이**
Vercel 설정만 바꾸면 됩니다:

1. Vercel → 프로젝트 → **Settings → Environment Variables**
2. 새 변수 추가: 이름 `GEMINI_MODEL`, 값은 새 모델 이름
   (예: `gemini-2.5-flash`. 최신 이름은 aistudio.google.com에서 확인)
3. **Redeploy**

이렇게 모델 이름을 코드 밖으로 빼두었기 때문에, 챗봇이 오래도록
망가지지 않고 유지될 수 있습니다.

  미리 점검하면 청년들이 먼저 발견하기 전에 고칠 수 있습니다.

  안에서 공유하길 권합니다. 비밀번호 화면이 필요하면 추가해 드릴 수 있습니다.
- Gemini 무료 한도(하루 약 1,500건)는 매일 초기화되며 만료되지 않습니다.
